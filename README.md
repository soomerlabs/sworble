# sworbl 🌱

**Seven minutes to crack the word of the day.**

A daily word game: hunt theme words on a letter board for 7:00 — every word scores, the day's
six hidden **clues** bank as you find them — then at 0:00 the board morphs into a keyboard and
you get six Wordle-style guesses to crack the day's word. One round per day. Solve it or see
you tomorrow.

Live: https://soomerlabs.github.io/sworble/ · vanilla JS · zero runtime dependencies · GitHub Pages

## Architecture

One rule: **pure modules decide, the shell acts.**

```
index.html          the app shell (~6.3k-line component): rendering, input, side effects,
                    storage/sfx wiring. Has a section-map ToC at the top of its script.
support.js          the dc-runtime (GENERATED — see Provenance below). React 18 UMD host.
vendor/             self-hosted React/ReactDOM (CDN fallback only).

sworble-core.js     determinism kernel: seeded PRNG (mulberry32), hashSeed, dayKey, deal.
                    THE CONTRACT: same dailyKey → byte-identical board for every player.
sworble-seed.js     clue placement: crossing-path stamper + two-pass exactly-6 packer.
sworble-solver.js   board word-search / findability.
sworble-daily.js    day content + guess rules: parseEntry, clueFor (extension banking),
                    scoreGuess (dup-safe Wordle colors), applySworbGuess, nextSlots.
sworble-status.js   THE day-state selector: dailyStatus, dayState, scoring reconciliation.
sworble-flow.js     round-lifecycle DECISIONS (start/resume/finale routing, pause legality).
sworble-run.js      run snapshots (serialize/validate/restore), count-in table, remainingSecs.
sworble-store.js    localStorage key registry + safe accessors (in-memory fallback).
sworble-net.js      backend client: typed errors, retry/backoff, durable offline outbox.
                    Unconfigured → fully local. Backend plan: docs/backend/SUPABASE-PLAN.md.

dailies.json        the daily content (sworb + theme words + definition per day).
dictionary.txt      validation dictionary (135k words, bundled — no third-party fetch).
words.js            starter tier: instant/offline vocabulary + "recognizable words" bias.
```

Every `sworble-*.js` module is pure — no DOM, no storage, no `this` — dual-exported for Node
tests, and deterministic (all randomness is injected seeded RNG).

## Run / develop

```
python3 -m http.server 8791     # serve — it's a static site; open http://localhost:8791
npm ci && npm test              # lint (eslint) + 12 suites, incl. a headless test that plays
                                # a full day (deal → spell → finale → solve → bank)
node tests/seed-stress.js       # board-packing stress harness (optional, not in npm test)
```

Deploying: see `docs/DEPLOY.md` (build stamp + cache-bust discipline). CI (`.github/workflows/
test.yml`) is the only gate between a push and production.

## Content

Days live in `dailies.json`, validated by `tests/dailies-check.js` (dictionary membership,
lengths — hints 4–7 with ≤2 sevens, sworb 5–7 — no prefix-pairs, and the exactly-6 packing
guarantee). Generate new days with any AI using `docs/content/GENERATION-PROMPT.md`, then run
the checker. Author ≥2 days ahead of UTC+14.

## Provenance

`support.js` is generated by Anthropic's dc-runtime (the app began life as a Claude artifact);
its generator source is not in this repo. It is stable, vendored, and treated as a black-box
host — application logic never lives there. Design history: `docs/superpowers/` (specs, plans,
and the audit trail of the 2026-07 one-game pivot).
